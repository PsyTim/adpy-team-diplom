Здесь будет описание диаграммы

1. таблица users
   В этой таблице содержатся пользователи бота
   Поля:
   1. id - первичный ключ
   2. vk_id - идентификатор вконтакте
   3. gender - пол
   4. birthday - дата рождения
   5. city_id - идентификатор города
   6. city - название города
   7. last_visit - последнее взаимодействте с ботом
   8. filter_age_from - настройки фильтра для поиска (минимальный возраст кандидата)
   9. filter_age_to - настройки фильтра для поиска (максимальный возраст кандидата)
   10. filter_gender - настройки фильтра для поиска (пол)
   11. filter_city_id - настройки фильтра для поиска (id города)
   12. filter_city - настройки фильтра для поиска (название города)
        Возможны последующие дополнения, изменения

2. users-profiles
   Эта таблица организует связь многие ко многим между таблицей пользователелей бота и таблицей анкет
   Поля:
   1. id - первичный ключ
   2. user_id - внешний ключ для таблицы пользователей
   3. profile_id - внешний ключ для таблицы профилей
   4. viewed - флаг содержащий дату и время, когда профиль был просмотрен либо NULL
   5. favorit - флаг содержащий дату и время, когда профиль был добавлен в избранное либо NULL
   6. blacklisted - флаг содержащий дату и время, когда профиль был добавлен в игнор либо NULL
        Возможны последующие дополнения, изменения

3. profiles

   3. gender - пол
   4. birthday - дата рождения
   5. city_id - идентификатор города
   6. city - название города
        Возможны последующие дополнения, изменения
